# Beruto `main.py` 和 CLI 的“串线图”
beruto项目完成后的运行不会涉及到具体的模块，而是全部封装起来，之后通过运行main.py来执行
1. **入口main像“电源按钮”**：在仓库根目录执行 `python src/main.py` 就像按下电源，`main.py` 会先把整个项目路径塞进 Python 的“查找列表”，这样后面需要的文件都找得到。
2. **把控制权交给“小前台CLI”**：`main.py` 随后把工作交给 `src/system/CLI.py` 的 `main()`。
3. **CLI 如何调接口**：`CLI.py` 里有个 `interactive_loop()`，会反复读取你的输入：
	- 输入 `help`、`h`、`?` 会输出帮助；
	- 输入 `exit`、`quit`、`q` 会结束程序；
	- 输入 `backtest`、`bt` 等命令时，会进入对应的分支；在这些分支里就能调用回测、拉数据等具体接口（目前只是占位提示，可按需求添加真实逻辑）。
4. **VS Code 小技巧**：想看某个函数的实现，右键函数名（比如 `interactive_loop`）→“转到定义”。编辑器会直接带你跳到函数所在文件/位置，非常适合快速追踪调用链。
5. **扩展方式**：想新增一条命令，就在 `interactive_loop()` 的 `if cmd == ...` 分支里写上你需要的处理流程，把真实的接口或脚本函数塞进去即可。

> 总之：`main.py` 负责“开机+交接”，而 `CLI.py` 像客服中心，集中处理用户命令并按需调用后端接口。通过右键跳转等技巧，即使没写过代码，也能跟着函数名称找到它们的来源和逻辑。
